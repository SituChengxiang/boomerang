# Utils

针对这样大噪声的场景做出的物理计算优化，这里东西有点多了，列个文档明确一下

1. `dataIO.py` 数据读取和写入、判断数据量是否足够，不依赖于自己写的模块
2. `standardization.py` 对于传入的序列进行时间标准化，不依赖于自己写的模块
   1. 保证时间严格递增
   2. 清理重复时间
   3. 开始的时间拉回0
   4. 三次样条插值对齐
3. `derivatives.py` 求导，依赖`scipy`，不依赖于自己写的模块，提供纯数值计算，因为噪声很大，不采用~~有限差分~~，提供的方法包括：
   1. 解析三次样条导数（处理整体轨迹->速度）
   2. Savitzky-Golay (SG) 滤波二阶导（样条插值分段线性，求加速度会好一点。建议把窗口长度设置为数据点的5%-10%）
   3. 基于“状态空间”的估算（RTS Smoother回传参数，修正开头发射角度、速度）
   4. 高斯回归统计（？）
4. `smoother.py` RTS Smoother 双向滤波平滑，依赖`pykalman`，不确定是否依赖自己写的模块
5. `filter.py` ：kalman滤波、低通滤波，不确定是否依赖自己写的模块
6. `physicsCal.py`: 依赖`filter`,`smoother`,`derivatives`，从位置数据出发，计算速度、机械能、势能和总能量以及判断能量变化
7. `visualize.py` 可视化模块，单独一个
